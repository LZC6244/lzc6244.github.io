## 简介
本技术文档围绕站点“搜索用户界面交互”的实现，系统性解析以下能力：
- 全屏弹窗的打开/关闭逻辑、事件监听与状态管理
- 主搜索框与弹窗内搜索框的双向同步机制
- 分页加载（滚动监听、动态追加、性能优化）
- 结果高亮显示（HTML 标签插入与样式应用）
- 键盘导航支持（Tab 聚焦等无障碍体验）
- 动画效果（淡入淡出过渡）
- CSS 样式定制与响应式设计适配方案
- 移动端适配与触摸事件处理细节

## 项目结构
搜索功能由三部分构成：
- 前端脚本：负责 DOM 操作、事件绑定、搜索算法、渲染与分页
- 样式表：定义搜索容器、弹窗、结果项、高亮、暗色模式与响应式布局
- 数据源：Jekyll 生成的静态 JSON 索引，供前端本地检索

## 详细组件分析

### 全屏弹窗：打开/关闭逻辑与状态管理
- 打开弹窗
  - 添加 .open 类以触发淡入动画
  - 记录当前滚动位置并固定 body，防止背景滚动
  - 确保弹窗内搜索栏存在，并将主输入框的值同步到弹窗输入框，自动聚焦
  - 设置打开守卫时间窗口，避免刚打开时被误判为点击遮罩而关闭
- 关闭弹窗
  - 移除 .open 类，恢复 body 滚动位置
- 遮罩关闭策略
  - 仅在 mousedown 发生在遮罩上且 click 时也落在遮罩上才关闭
  - 若用户有选中文本则不关闭，避免打断阅读
- 关闭按钮
  - 独立按钮点击直接关闭

### 分页加载：滚动监听、动态加载与性能优化
- 分页参数
  - PAGE_SIZE=8，维护 currentResults、currentPage、isLoading、allLoaded 等状态
- 滚动监听
  - 在结果面板首次创建时绑定 scroll 事件
  - 当 scrollTop + clientHeight 接近 scrollHeight 时触发 loadMore
- 动态加载
  - 使用 requestAnimationFrame 批量渲染，减少重排
  - 使用 DocumentFragment 一次性插入节点
  - 首屏显示“加载中…”，末页显示“已加载全部 N 篇”
  - 若内容未填满面板，自动继续加载下一页
- 性能优化
  - 防抖输入，避免频繁搜索
  - 预加载索引，提升首开速度
  - 结果去重，避免重复条目
  - 仅对可见部分进行 DOM 操作

```mermaid
flowchart TD
Enter(["loadMore 入口"]) --> CheckState{"是否正在加载/已全部加载?"}
CheckState --> |是| Exit(["退出"])
CheckState --> |否| SetLoading["设置 isLoading=true"]
SetLoading --> CalcRange["计算 start/end 切片范围"]
CalcRange --> FirstPage{"是否第一页?"}
FirstPage --> |是| ShowLoading["插入 '加载中...' 提示"]
FirstPage --> |否| SkipLoad["跳过提示"]
ShowLoading --> RAF["requestAnimationFrame 渲染"]
SkipLoad --> RAF
RAF --> Slice["items = results.slice(start, end)"]
Slice --> Fragment["DocumentFragment 构建节点"]
Fragment --> Append["append 到面板"]
Append --> UpdateState["currentPage++ / isLoading=false"]
UpdateState --> AllDone{"是否已加载完?"}
AllDone --> |是| ShowEnd["显示 '已加载全部' 提示"]
AllDone --> |否| AutoNext{"内容未填满面板?"}
AutoNext --> |是| Recur["递归调用 loadMore()"]
AutoNext --> |否| Exit
ShowEnd --> Exit
```

## 故障排查指南
- 无法加载搜索索引
  - 现象：面板显示错误提示
  - 可能原因：search.json 路径不可达或格式异常
  - 定位：检查 fetch 失败分支与错误提示渲染
- 弹窗无法关闭
  - 现象：点击遮罩无效
  - 可能原因：用户存在选中文本导致关闭被阻止
  - 定位：检查选中检测逻辑与遮罩点击分支
- 滚动加载不触发
  - 现象：滚动到底部不再加载
  - 可能原因：isLoading/allLoaded 状态异常或面板高度计算问题
  - 定位：检查滚动监听与 loadMore 条件
- 高亮异常
  - 现象：关键词未被高亮或高亮错乱
  - 可能原因：正则转义或中英文匹配分支问题
  - 定位：检查 escapeRegex 与 keywordMatches/highlightMatch 逻辑
