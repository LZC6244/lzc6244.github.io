按 Jekyll 约定目录组织，职责分层清晰：
- `_layouts/`：定义 `home.html`（分类/日期双视图归档，通过 JS IIFE 切换）和 `post.html`（文章详情 + Schema.org 标记 + 目录侧栏 + 代码块工具栏），二者均通过 front matter `layout: default` 继承默认布局。
- `_includes/`：拆出 `head.html`（SEO、Google Fonts、favicon、analytics）、`header.html`（站点标题 + 搜索框）、`footer.html`、`disqus_comments.html`（条件加载 Disqus），供布局按需 include。
- `_plugins/`：纯 Ruby 脚本，全部通过 `Jekyll::Hooks.register <scope>, :<phase>` 在构建生命周期注入，不暴露 Liquid 标签，仅作为钩子副作用运行——`check_missing_images.rb` 扫描 Markdown/HTML 图片路径并警告缺失文件；`escape_code_liquid.rb` 在 `posts.pre_render` 阶段自动为围栏/行内/HTML code 中的 `{{ }}` 包裹 `{% raw %}`，并将有序列表内的 ``` 替换为 ~~~；`normalize_code_lang.rb`、`year_category_filter.rb`、`ruby34_compat.rb` 分别做语言规范化、年份过滤兼容及 Ruby 3.4+ `String#untaint` 回退。
依赖方向单向：布局 → includes；插件只读 `site.posts/docs` 或修改 `post.content/post.data`，不反向引用 HTML。
