本模块由两类产物组成：
- Jekyll 数据源：`pages/feed.xml` 与 `search.json` 使用 YAML Front Matter（`layout: null`）声明，构建时由 Jekyll 遍历 `site.posts` 生成 RSS 2.0 与可被前端全文检索的 JSON 数组；`search.json` 通过 split `<pre>` 块并 strip_html/strip_urls 过滤代码块与链接，仅保留正文片段。
- 独立静态 HTML：`file-viewer.html` 与 `wechat_share.html` 不使用 Jekyll 布局，前者通过 URL 参数 `?file=` 动态 fetch 文本资源并以 `<pre><code>` 展示，支持下载与暗色模式；后者借助 qrcodejs 将 `?url=` 渲染为二维码，用于微信分享场景。
依赖方向单向：Jekyll 在构建期消费 `site.posts`，运行时页面只消费生成的静态产物，无后端交互。
