模块按 Jekyll 约定目录组织：
- `_layouts/`：定义 `home.html`（分类/日期双视图归档）和 `post.html`（文章详情 + 目录侧栏 + 代码块工具栏），二者均通过 front matter `layout: default` 继承默认布局。
- `_includes/`：拆出 `head.html`（SEO、字体、favicon、analytics）、`header.html`（站点标题 + 搜索框）、`footer.html`（联系信息 + 社交链接 + 搜索弹窗遮罩）、`disqus_comments.html`（条件加载 Disqus），供布局按需 include。
- `_plugins/`：纯 Ruby 脚本，全部通过 `Jekyll::Hooks.register` 在构建生命周期注入，不暴露 Liquid 标签，仅作为钩子副作用运行。
依赖方向单向：布局 → includes；插件只读 `site.posts/docs` 或修改 `post.content/post.data`，不反向引用 HTML。
