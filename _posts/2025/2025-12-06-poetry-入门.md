# 安装

[linux/mac] 使用 pyenv 安装 python 环境（切换 python 版本），使用 poetry 管理安装 python 依赖

pyenv 安装参考 [pyenv 入门]({% post_url 2025/2025-12-13-pyenv-入门 %})

```Bash
# 安装 poetry
curl -sSL https://install.python-poetry.org | python3 -

# 确认 poetry 的安装路径
# 在安装 Poetry 时，它默认会将可执行文件安装到 $HOME/.local/bin 目录下
ls -lh ~/.local/bin/poetry
 
# 在 ~/.bashrc 末尾添加以下 poetry 配置，为其配置环境变量（根据实际情况更改）
export PATH="/root/.local/pipx/venvs/poetry/bin:$PATH" 
 
# 应用 ~/.bashrc 配置
source ~/.bashrc
 
# 为 bash 终端类型配置 tab 自动补全 poetry 命令，其他如 zsh 请参考官方文档
poetry completions bash >> ~/.bash_completion
# 将 poetry 创建的 python 虚拟环境路径配置为项目路径下的 .venv
poetry config virtualenvs.in-project true
# 对项目进行 poetry 的初始化（创建pyproject.toml文件），此处无需进行，毕昇官方已进行
poetry init
# 为 poetry 安装 python 依赖配置国内清华源（仅本项目）
poetry source add --priority=primary tsinghua https://pypi.tuna.tsinghua.edu.cn/simple/
# 为当前项目创建 python3.10 虚拟环境（或者此处可传入具体 python 路径）
poetry env use 3.10
# (可选) poetry install 如果卡住卡顿可以配置此选项不启用系统密钥环来存储凭据
# 实测有时影响很大，猜测是在界面的操作系统会弹窗，但是使用 ssh 终端登入时无法发现处理
poetry config keyring.enabled false
# 安装项目 python 依赖
poetry install
```



Windows 下的 pyenv 和 poetry 请参考下方二者的官方链接进行安装

# poetry 发布到 pypi



```Plain Text
# 生成 whl 和 tar.gz
poetry build

# 发布到 pypi
poetry publish -u __token__（用户名） -p __token__（实际token）
```
